# 🌐 WebGL Playtest Guide

Two ways to get Hillbilly Alien Shooter running in a browser. **Option A is the
fastest way to playtest today** (5 minutes, zero account setup). **Option B**
gives you an always-up-to-date public URL rebuilt automatically on every merge
to `main` — it needs a one-time free Unity license setup.

---

## Option A — Local WebGL build (fastest)

From your Unity project (with the repo's `Assets/_Project` in place):

1. In Unity: **File ▸ Build Profiles** (or Build Settings) → select **Web /
   WebGL** → **Switch Platform**. First switch takes a few minutes.
   > If WebGL isn't listed, add the **WebGL Build Support** module to your
   > editor via Unity Hub → Installs → ⚙ → Add modules.
2. Menu: **Tools ▸ Hillbilly ▸ Build WebGL** — one click. It rebuilds the farm
   scene, applies browser-friendly settings (Gzip + decompression fallback),
   and outputs to `build/WebGL/`.
3. **Playtest it.** Browsers refuse WebGL from `file://`, so serve the folder:
   ```bash
   cd build/WebGL
   python -m http.server 8000
   ```
   Open <http://localhost:8000>. (Any static server works; `npx serve` too.)

### Sharing with friends: itch.io (free, ~2 minutes)
1. Zip the **contents** of `build/WebGL/` (index.html must be at the zip root).
2. On itch.io: Create new project → *Kind of project: HTML* → upload the zip →
   check **"This file will be played in the browser"** → set viewport
   ~1280×720 → Save & view. Done — share the link.

## Option B — Auto-deploy to GitHub Pages (CI)

Once set up, every merge to `main` rebuilds the game and publishes it at:

**➡ https://newtonnowatzke-tech.github.io/hillbilly-alien-shooter/**

The workflow is already in the repo (`.github/workflows/webgl-deploy.yml`).
It needs two one-time switches flipped:

### Step 1 — Enable GitHub Pages
Repo **Settings ▸ Pages ▸ Build and deployment ▸ Source** → select
**"GitHub Actions"**.

### Step 2 — Add your (free) Unity license secrets
GameCI builds with a real Unity editor in CI, which must be activated with
*your* Unity Personal license (free). Add three repository secrets under
**Settings ▸ Secrets and variables ▸ Actions ▸ New repository secret**:

| Secret | Value |
|---|---|
| `UNITY_EMAIL` | your Unity account email |
| `UNITY_PASSWORD` | your Unity account password |
| `UNITY_LICENSE` | the full text contents of your `Unity_lic.ulf` file (see below) |

**Finding your `Unity_lic.ulf`** (it may already exist on your machine):

| OS | Path |
|---|---|
| Windows | `C:\ProgramData\Unity\Unity_lic.ulf` |
| macOS | `/Library/Application Support/Unity/Unity_lic.ulf` |
| Linux | `~/.local/share/unity3d/Unity/Unity_lic.ulf` |

Open it in a text editor, copy **everything**, paste as the `UNITY_LICENSE`
secret.

**Can't find the file?** (Newer Unity Hub licenses sometimes don't create one.)
Use the included helper: **Actions tab ▸ "Request Unity Activation File" ▸ Run
workflow**, download the `.alf` artifact, upload it at
<https://license.unity3d.com/manual>, pick *Unity Personal*, download the
resulting `.ulf`, and paste its contents into the `UNITY_LICENSE` secret.

### Step 3 — Run it
**Actions ▸ "WebGL Build & Deploy" ▸ Run workflow** (or just merge to `main`).
First build takes ~15–25 min (the editor image is big); later runs are faster
thanks to the Library cache. The play URL appears on the deploy job when done.

## How CI builds from a scripts-only repo

This repo intentionally contains no binary Unity assets — the scene and all
data assets are **generated by code**. The CI pipeline
(`WebGLBuilder.BuildWebGL`) makes that work on a fresh checkout:

1. `Packages/manifest.json` + `ProjectSettings/ProjectVersion.txt` tell Unity
   which packages/editor to use (URP, Input System, uGUI on Unity 6 LTS).
2. The builder flips **Active Input Handling → Both** (fresh projects default
   to the legacy input manager, which would break the New Input System code).
3. It creates and assigns a **URP pipeline asset** if none exists (otherwise
   URP-shader materials render pink).
4. `FarmSceneBuilder.BuildFarmSceneHeadless()` regenerates `Farm.unity` and the
   ScriptableObject data — identical to the in-editor menu item.
5. `BuildPipeline.BuildPlayer` produces the WebGL build with **Gzip +
   decompression fallback**, so it runs on GitHub Pages/itch.io without any
   server header configuration.

> ⚠️ The `Packages/` and `ProjectSettings/` folders in this repo exist **for CI
> only**. When setting up your local project, keep copying just
> `Assets/_Project` as described in [SETUP.md](../SETUP.md) — don't overwrite
> your local project's own settings with these.

## Browser-specific behaviour (already handled in code)

- **Pointer lock needs a click** — click the game once and the mouse captures;
  firing that first click is normal.
- **Esc** exits pointer lock in browsers; the game detects the lost lock and
  **auto-pauses**. **P** also opens the pause menu directly.
- The **Quit** button is hidden on WebGL (browsers don't allow it).

## Troubleshooting

| Symptom | Fix |
|---|---|
| Workflow fails at "Check Unity license secrets" | Add the three secrets from Step 2. |
| Workflow fails during activation | `UNITY_LICENSE` must be the *entire* `.ulf` file text; email/password must match the account that owns the license. |
| 404 at the Pages URL | Settings ▸ Pages ▸ Source must be "GitHub Actions"; check the deploy job succeeded. |
| Black page / loader hangs locally | You opened `index.html` via `file://` — serve over HTTP (see Option A step 3). |
| Pink materials in the build | The URP pipeline wasn't assigned; use **Tools ▸ Hillbilly ▸ Build WebGL** (it self-heals this) rather than a raw Build Settings build on a project without URP configured. |
| CI editor version mismatch warning | Edit `ProjectSettings/ProjectVersion.txt` to any Unity `6000.0.x` you prefer; the repo has no serialized assets, so versions are interchangeable. |
